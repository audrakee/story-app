## DESKRIPSI APLIKASI

Story App adalah Progressive Web Application (PWA) yang memungkinkan pengguna untuk berbagi cerita mereka dengan foto dan informasi lokasi. Aplikasi ini dibangun dengan vanilla JavaScript dan mengimplementasikan semua kriteria submission proyek kedua Dicoding.

## FITUR UTAMA

### 1. Authentication (Login & Register)
- Pengguna dapat mendaftar dengan nama, email, dan password (min 8 karakter)
- Pengguna dapat login dan mendapatkan JWT token
- Token disimpan di localStorage untuk manajemen session

### 2. SPA (Single Page Application) dengan Transisi Halaman
- Implementasi hash routing (#/) untuk navigasi tanpa reload halaman
- View transitions dengan animasi fadeInUp
- Arsitektur MVP dengan pemisahan concerns (db.js, api.js, utils.js, router.js, auth.js, notification.js)
- Halaman terpisah untuk: Login, Register, Home, Add Story, Detail Story, dan Favorites

### 3. Menampilkan Data dan Marker pada Peta
- Integrasi Leaflet.js untuk peta interaktif
- Menampilkan stories dalam bentuk daftar dengan gambar, nama penulis, deskripsi, dan lokasi
- Visualisasi lokasi stories menggunakan markers pada peta
- Popup interaktif pada marker untuk melihat detail story
- Multiple tile layers (OpenStreetMap dan Satellite)
- Click pada marker untuk membuka detail story

### 4. Fitur Tambah Data Baru (Add Story)
- Form untuk input deskripsi story
- Upload foto dari device atau kamera langsung
- Pemilihan koordinat latitude/longitude dengan click pada peta
- Preview gambar sebelum upload
- Validasi file size (max 1MB)
- Fitur kamera langsung dengan media stream
- Pengiriman data ke API secara asynchronous

### 5. IndexedDB Integration
- Menyimpan stories yang sudah diambil dari API di IndexedDB
- Fitur Favorites untuk menandai stories favorit
- Fitur Pending Uploads untuk stories yang ditambah saat offline
- CRUD operations: Create, Read, Delete

### 6. PWA (Progressive Web App)
- Installable di mobile dan desktop
- Service Worker untuk offline support
- Caching strategy: Cache First untuk static assets, Network First untuk API
- Manifest.json dengan icon dan screenshots
- Install prompt untuk mengajak user install app
- Support untuk offline viewing (app shell dan cached stories)

### 7. Push Notifications
- Subscribe/unsubscribe ke web push notifications

- Notifikasi dinamis dari server
- Button toggle untuk enable/disable notifications
- Action untuk menutup notifikasi

### 8. Accessibility (WCAG)
- Semantic HTML elements (header, nav, main, footer, article)
- Alt text pada semua gambar
- Labels pada semua input elements
- ARIA labels dan attributes
- Keyboard navigation support (focus-visible)

- Responsive design (375px mobile, 768px tablet, 1024px desktop)

### 9. Offline Support
- Caching API responses di service worker
- IndexedDB untuk local storage
- Pending uploads sync saat kembali online

## Menjalankan secara Lokal (untuk reviewer)

1. Clone repository atau download project
2. Install dependencies:

```
npm install
```

3. Jalankan server lokal (server wajib untuk service worker):

```
npm start
```

4. Buka di browser: http://localhost:8080

Catatan: `package.json` dan `package-lock.json` sudah disertakan agar reviewer dapat menjalankan `npm install` untuk memasang dependency (`http-server`).

Untuk deploy sendiri:
1. Buat repository di GitHub
2. Push semua file ke repository
3. Enable GitHub Pages di settings
4. Deploy URL akan tersedia di settings

## API YANG DIGUNAKAN

### Dicoding Story API
- Base URL: https://story-api.dicoding.dev/v1
- Endpoints:
  - POST /register - Register user baru
  - POST /login - Login dan dapatkan token
  - GET /stories - Dapatkan semua stories
  - GET /stories/:id - Dapatkan detail story
  - POST /stories - Tambah story (authenticated)
  - POST /notifications/subscribe - Subscribe push notification
  - DELETE /notifications/subscribe - Unsubscribe push notification

### Authentication
- Menggunakan JWT token di header: Authorization: Bearer <token>
- Token disimpan di localStorage setelah login
